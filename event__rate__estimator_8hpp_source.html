00012  *
00013  * When designing the time delay analysis system it is interesting to
00014  * answer questions such as these:
00015  * - How many messages per second arrive on average?  How many per
00016  * millisecond?
00017  * - More generally, what is the distribution of messages per second,
00018  * millisecond and microsecond?

00020  * This information will define the performance requirements of the
00021  * system, help us define simulators for the environment, define
00022  * performance requirements and ultimately measure the performance of
00023  * the system.
00024  *
00025  * We are going to need to measure such statistics per-symbol, there
00026  * are roughly 15,000 symbols in the US market, and we would like to
00027  * keep memory utilization for this data to be below 8GiB [1].  That
00028  * leaves us with roughly 600 KiB per symbol.  So we cannot be just
00029  * wasteful with memory.
00030  *
00031  * Assume you are interested in statistics about messages per
00032  * millisecond.  In that scenario, this class keeps exactly 1000
00033  * buckets, representing the last 1000 microseconds or the last
00034  * rolling millisecond.  Each bucket contains a counter representing
00035  * the number of events in that microsecond.
00036  *
00037  * If a new sample arrives for the current microsecond we simply
00038  * increment the counter.  If a new sample arrives for the next
00039  * microsecond we (1) count the number of events in the current
00040  * buckets, (2) update the statistics, (3) "rotate" the circular
00041  * buffer by reseting the oldest buffer to 0 and moving the time
00042  * pointer, and (4) insert the new timestamp in the current bucket. 
00043  * If the new sample skips one more more microseconds then steps 1-3
00044  * are repeated until the circular buffer catches up to the new
00045  * timestamp, and then the counter is incremented.
00046  *
00047  * [1] This is an arbitrary limit based on the memory on coryan@
00048  * workstation (16 GiB).  I decided that only 1/2 of the memory would
00049  * be dedicated to this application.
00050  *
00051  * @tparam duration_type Defines the size of the buckets.
00052  */
