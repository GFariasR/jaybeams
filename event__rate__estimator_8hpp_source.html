10 /**
11  * Helper class to compute statistics about messages frequency.
12  *
13  * When designing the time delay analysis system it is interesting to
14  * answer questions such as these:
15  * - How many messages per second arrive on average? How many per
16  * millisecond?
17  * - More generally, what is the distribution of messages per second,
18  * millisecond and microsecond?
19  *
20  * This information will define the performance requirements of the
21  * system, help us define simulators for the environment, define
22  * performance requirements and ultimately measure the performance of
23  * the system.
24  *
25  * We are going to need to measure such statistics per-symbol, there
26  * are roughly 15,000 symbols in the US market, and we would like to
27  * keep memory utilization for this data to be below 8GiB [1]. That
28  * leaves us with roughly 600 KiB per symbol. So we cannot be just
29  * wasteful with memory.
30  *
31  * Assume you are interested in statistics about messages per
32  * millisecond. In that scenario, this class keeps exactly 1000
33  * buckets, representing the last 1000 microseconds or the last
34  * rolling millisecond. Each bucket contains a counter representing
35  * the number of events in that microsecond.
36  *
37  * If a new sample arrives for the current microsecond we simply
38  * increment the counter. If a new sample arrives for the next
39  * microsecond we (1) count the number of events in the current
40  * buckets, (2) update the statistics, (3) "rotate" the circular
41  * buffer by reseting the oldest buffer to 0 and moving the time
42  * pointer, and (4) insert the new timestamp in the current bucket.
43  * If the new sample skips one more more microseconds then steps 1-3
44  * are repeated until the circular buffer catches up to the new
45  * timestamp, and then the counter is incremented.
46  *
47  * [1] This is an arbitrary limit based on the memory on coryan@
48  * workstation (16 GiB). I decided that only 1/2 of the memory would
49  * be dedicated to this application.
50  *
51  * @tparam duration_type Defines the size of the buckets.
52  */
