00016  * A histogram class with controllable binning and range strategy.
00017  *
00018  * We are interested in capturing histograms of latency, and of rate
00019  * measurements, and other things.  The challenge is that we need good
00020  * precision, but we also need to keep thousands of these histograms
00021  * in memory.  To increase precision in histograms one must increase
00022  * the number of bins, but that increases the memory requirements.
00023  *
00024  * Consider, for example, a typical latency measurement that typically
00025  * ranges from 1 to 400 microseconds, but sometimes can peak to
00026  * seconds.  Capturing the full range at microsecond precision would
00027  * require MiB of memory.  While this is not a problem for a single
00028  * histogram, what if we want to keep this histogram per symbol?
00029  *
00030  * As an alternative, we can keep this histogram at full resolution
00031  * between 0 and 1,000 microseconds, drop to 10 microsecond bins from
00032  * there to 10,000 microseconds, then increase to 100 microsecond bins
00033  * and so forth.  For values higher than a prescribed limit we simply
00034  * record "overflow".
00035  *
00036  * This class implements such a histogram, with a user-provided
00037  * mapping strategy to define the bins.
00038  *
00039  * @tparam binning_strategy_t Please see @ref jb::binning_strategy_concept.
00040  * @tparam counter_type The type used for the counters.  In most
00041  *   applications using int is sufficient, but if you are counting
00042  *   really large numbers you might consider using std::uint64_t, or
00043  *   if you are memory constrained consider using std::int16_t.

00125    * The inverse of the accumulated density function.  Find the
00126    * smallest value Q such that at most q * nsamples of the samples

00149           observed_min(), binning_.histogram_min(), y_a, s, q);
