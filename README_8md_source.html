00008 The JayBeams library performs relative time delay estimation of market
00009 feeds.  That is, given two feeds for the same inside quote data it
00010 estimates, in near real-time which one is faster, and by how much.

00015 ### Motivation
00016 
00017 The US equity markets have a lot of redundant feeds with basically the
00018 same information, or where one feed is a super set of a second set.
00019 When one has two feeds an interesting question is to know how much
00020 faster is one feed vs. the other, or rather, how much faster it is
00021 right now, because the latency changes over time.  There are (usually)
00022 no message IDs or any other simple way to match events in one feed
00023 vs. events in the second feed.  So the problem of "time-delay
00024 estimation" requires some heavier computation, and doing this in
00025 real-time is extra challenging.  The current implementation is based
00026 on cross-correlation of the two signals, using FFTs on GPUs to
00027 efficiently implement the cross-correlations.
00028 
00029 #### Really, that is the movation?

00031 I confess, I wanted to learn how to program GPUs, and given my
00032 background this appeared as an interesting application.

00040 ### What is with the name?
00041 
00042 JayBeams is named after a [WWII electronic warfare](https://en.wikipedia.org/wiki/List_of_World_War_II_electronic_warfare_equipment) system.
00043 The name was selected more or less at random from the Wikipedia list
00044 of such systems, and is not meant to represent anything in
00045 particular.  It sounds cool, in a Flash Gordon kind of way.
