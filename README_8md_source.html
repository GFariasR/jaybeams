6 Another project to have fun coding.
7 
8 The JayBeams library performs relative time delay estimation of market
9 feeds. That is, given two feeds for the same inside quote data it
10 estimates, in near real-time which one is faster, and by how much.
11 
12 - Licensing details are found in the LICENSE file.
13 - The installation instructions are in the INSTALL file.
14 
15 ### Motivation
16 
17 The US equity markets have a lot of redundant feeds with basically the
18 same information, or where one feed is a super set of a second set.
19 When one has two feeds an interesting question is to know how much
20 faster is one feed vs. the other, or rather, how much faster it is
21 right now, because the latency changes over time. There are (usually)
22 no message IDs or any other simple way to match events in one feed
23 vs. events in the second feed. So the problem of "time-delay
24 estimation" requires some heavier computation, and doing this in
25 real-time is extra challenging. The current implementation is based
26 on cross-correlation of the two signals, using FFTs on GPUs to
27 efficiently implement the cross-correlations.
28 
29 #### Really, that is the movation?
30 
31 I confess, I wanted to learn how to program GPUs, and given my
32 background this appeared as an interesting application.
33 
34 #### So where is the code?

36 I am pushing the code slowly. I want to make sure it compiles and
37 passes its tests on at least a couple of Linux variants. This can be
38 a challenge given the dependency on OpenCL libraries.

40 ### What is with the name?
41 
42 JayBeams is named after a [WWII electronic warfare](https://en.wikipedia.org/wiki/List_of_World_War_II_electronic_warfare_equipment) system.
43 The name was selected more or less at random from the Wikipedia list
44 of such systems, and is not meant to represent anything in
45 particular. It sounds cool, in a Flash Gordon kind of way.
